# Pilot Session Checklist

> **용도:** 6세션 수집 시 매 세션마다 따라가는 절차 체크리스트  
> **인쇄해서 옆에 두고 쓰거나, 노션/메모장에 복사해서 체크하며 진행**

---

## 세션 전 (2분)

- [ ] 해당 모델 사이트 접속 (claude.ai / chatgpt.com / gemini.google.com)
- [ ] **모델 자체 인코그니토/임시 채팅 모드 켜기:**
  - Claude: 인코그니토 대화 (새 채팅에서 토글)
  - ChatGPT: Temporary Chat (설정에서 활성화)
  - Gemini: Gemini 앱 활동 끄기 (활동 설정 확인)
- [ ] 모델 버전 확인 + 기록 (예: Claude Opus 4.6, GPT-5.4 등)
- [ ] temperature / 기타 설정 기본값인지 확인
- [ ] 프롬프트 파일 열어두기 (P1 또는 P2 multiturn.md)
- [ ] ⚠️ **모델 인코그니토 = 대화 저장 안 됨. 창 닫기 전 PDF 저장 필수!**

---

## 세션 진행 (15~20분)

### STEP 1: 시스템 프롬프트

- [ ] 시스템 프롬프트 전문 복붙 → 전송
- [ ] 모델 준비 응답 확인 (P1: "네, 시작할게요" / P2: "네")
- [ ] **준비 응답이 이상하면 기록** (예: 장황하게 설명하거나 캐릭터를 벗어남)

- [ ] 턴 10 복붙 → 응답 확인
- [ ] 턴 11 복붙 → 응답 확인

**진행 중 주의사항:**
- 꼬리질문/추가질문 하지 않기 (프로빙 금지)
- 오타나 다른 텍스트 섞이지 않게 주의 (프롬프트 파일에서 코드블록 안의 내용만 복붙)
- 모델이 캐릭터를 깨고 "저는 AI입니다" 류의 응답을 하면 → 그 턴 기록 + 다음 턴 그냥 진행 (재시도 안 함)

## 세션 직후 — 창 닫기 전! (3분)

### 저장 1: PDF (원본 보존용)

- [ ] Ctrl+P (Mac: Cmd+P) → "PDF로 저장" 선택
- [ ] 파일명 규칙: `pilot_S{번호}_{페르소나}_{모델}.pdf`
  - 예: `pilot_S1_P1김도현_Claude.pdf`
  - 예: `pilot_S4_P2이수진_Claude.pdf`
- [ ] 저장 위치: 프로젝트 폴더 `responses/pilot/`

### 저장 2: 텍스트 복붙 (분석용)

- [ ] 대화 전체 선택 (Ctrl+A) → 복사 (Ctrl+C)
- [ ] `pilot_response_log_multiturn.md` 열기
- [ ] 해당 세션 섹션에 턴별로 응답 붙여넣기

### 메타데이터 기록

- [ ] 응답 로그 상단 메타데이터 테이블에 기록:
  - 모델 버전 (정확한 버전명)
  - 실행 일시 (YYYY-MM-DD HH:MM)
  - temperature (default면 "default")
  - 총 턴 수 (정상 14/14인지, 중간에 문제 있었는지)

### 이상 징후 메모 (해당 시)

- [ ] 캐릭터 붕괴 턴이 있었으면 턴 번호 + 내용 메모
- [ ] 할루시네이션 (없는 경험 생성) 발견 시 메모
- [ ] 기타 특이사항 메모

## 권장 실행 순서

한 페르소나를 3모델 연속으로 돌리면, 직전 모델 응답이 평가에 영향을 줄 수 있어.
모델을 섞어서 진행하는 게 순서 효과(order effect)를 줄여줌.

| 순서 | 세션 | 페르소나 | 모델 |
|------|------|----------|------|
| 1 | S1 | P1 김도현 | Claude Opus 4.6 |
| 2 | S4 | P2 이수진 | Claude Opus 4.6 |
| 3 | S2 | P1 김도현 | GPT-5.4 |
| 4 | S5 | P2 이수진 | GPT-5.4 |
| 5 | S3 | P1 김도현 | Gemini 3.1 Pro |
| 6 | S6 | P2 이수진 | Gemini 3.1 Pro |

이렇게 하면 같은 모델에서 P1→P2를 연속으로 보면서 "이 모델이 페르소나를 얼마나 다르게 연기하는지"를 바로 체감할 수 있어.
